MULTI-AGENT PATTERNS

6 Padrões de Orquestração

1
Pattern 1
Sequential
Cada agente executa em ordem fixa. Output alimenta o próximo. ShopAgent usa este.
2
Pattern 2
Parallel
Todos executam ao mesmo tempo. Resultado consolidado por merge no final.
3
Pattern 3
Hierarchical
Um planner central delega para specialists que executam e reportam.

Sequential é o mais comum em produção. Parallel quando velocidade importa. Hierarchical quando complexidade exige supervisão.

MULTI-AGENT PATTERNS

Inteligência de Decisão

4
Pattern 4
Routing
Classificação NL direciona a query para o agente especializado certo.
5
Pattern 5
Debate
Agentes independentes votam. A melhor resposta vence por consenso.
6
Pattern 6
Reflection
O agente revisa o próprio output e itera até atingir qualidade.

Routing é o cérebro, Debate para consenso, Reflection para qualidade. Estes 6 padrões combinados formam qualquer sistema multi-agente.
